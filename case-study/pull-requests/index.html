Case Study

Pull requests

One minute video

Provides more detail than the words with pictures

Context

  • My Dev team had no code review process.
  • Our code quality & knowledge on the codebase was lacking.
  • We were only using Azure Dev Ops for git repos.

What I did

  • I researched pull requests and code reviews.
  • Presented the idea to my teams.
  • Implement mandatory pull requests whenever a change to the codebase & measured results.

Results

  • 3 code quality issues found per pull request. *
  • * - On average | Code quality issue is an issue on readibility, or maintainability, e.g. comments or variable names.
